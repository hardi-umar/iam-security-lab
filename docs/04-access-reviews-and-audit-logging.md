# Lab 04 - Access Reviews and Audit Logging

## Objective

Demonstrate how periodic access reviews and audit logging help organizations maintain security, accountability, and compliance.

## Environment

- Microsoft Entra ID
- Azure Activity Logs
- Azure Sign-In Logs
- Microsoft Defender

## Scenario

An organization wants to verify that users maintain only the access required for their job functions and that administrative actions can be audited.

## Activities Performed

### Access Review

Reviewed user accounts and group memberships.

Validated:

- Active users
- Disabled users
- Administrative accounts
- Group assignments
- Role assignments

### Excessive Access Review

Identified users with unnecessary permissions.

Examples:

- Users assigned to multiple privileged groups
- Accounts with outdated access
- Users retaining access after role changes

### Audit Log Review

Reviewed:

- User creation events
- User deletion events
- Role assignment events
- Authentication events
- Administrative changes

### Documentation

Recorded findings and recommended corrective actions.

## Security Benefits

- Improves accountability
- Reduces privilege creep
- Supports compliance requirements
- Enhances visibility into administrative activity

## Security Concepts Demonstrated

- Access Governance
- Audit Logging
- Accountability
- Compliance Monitoring
- Privileged Access Review
- Identity Governance
